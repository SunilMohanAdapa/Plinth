{% blocktrans trimmed %} Monkeysphere can be used to authenticate visitors by using OpenPGP keys as HTTPS client certificates. {% endblocktrans %}

{% blocktrans trimmed %} A client intending to authenticate with {{ box_name }} using this method will need an OpenPGP public/private key pair. This key needs be signed by another key after verifying the identity of the client such as during a key signing party. The public key of the client and the signatures need to be published on a keyserver. Client then generates an X509 client certificate from this key pair and imports it into a browser. When using services of {{ box_name }}, browser submits this certificate automatically for verification and the client is verified without the need for a password. {% endblocktrans %}

{% blocktrans trimmed %} The signing key is known here as a certifier. A list of certifiers as managed here will be trusted 'ultimately'. When a client's certificate is received by {{ box_name }}, it is accepted as valid if any of the certifiers have signed the key. {% endblocktrans %}
